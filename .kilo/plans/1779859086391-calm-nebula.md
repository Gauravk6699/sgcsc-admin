# Certificate Generation Issue Resolution Plan

## Problem Analysis
Based on code review of marksheet-generator.js and certificate-generator.js:

### Compression Issue
Both generators use JPEG compression at 95% quality when converting canvas to PDF:
```javascript
pdf.addImage(_canvas.toDataURL('image/jpeg', 0.95), 'JPEG', 0, 0, W, H);
```
This 95% quality setting causes visible compression artifacts in generated PDFs.

### Zoom Issue
PDFs open at unexpected zoom levels likely due to:
- Using pixel units (`unit: 'px'`) in jsPDF without proper DPI consideration
- Canvas dimensions not matching expected print/display ratios

### QR Code Requirement
Need to add QR code generation and placement on each certificate.

## Solution Plan

### 1. Fix Compression Issue
**Action:** Increase JPEG quality to maximum or use lossless PNG format
- Change quality from 0.95 to 1.0 for maximum JPEG quality
- OR switch to PNG format for lossless compression (better for text/images)
- Update both `_canvasToPDF()` functions in both generator files

### 2. Fix Zoom Issue
**Action:** Adjust PDF generation to use proper scaling
- Consider using `unit: 'mm'` or `unit: 'pt'` with proper DPI conversion
- Or maintain pixel units but ensure canvas size matches intended display size
- Test with different zoom settings in PDF viewers

### 3. Add QR Code Functionality
**Action:** Implement QR code generation and placement
- Add QR code generation using a reliable method (could use external library or canvas-based generation)
- Add QR code field configuration to both generators
- Position QR code appropriately on certificates (likely corner area)
- Modify render functions to draw QR code when data is provided

## Implementation Steps

### Step 1: Fix Compression
In both generator files:
```javascript
// Change from:
pdf.addImage(_canvas.toDataURL('image/jpeg', 0.95), 'JPEG', 0, 0, W, H);
// To:
pdf.addImage(_canvas.toDataURL('image/png', 1.0), 'PNG', 0, 0, W, H);
// OR for maximum JPEG quality:
pdf.addImage(_canvas.toDataURL('image/jpeg', 1.0), 'JPEG', 0, 0, W, H);
```

### Step 2: Fix Zoom
Test different approaches:
- Option A: Keep pixels but ensure canvas matches display expectations
- Option B: Convert to mm: `unit: 'mm', format: [W*0.264583, H*0.264583]` (px to mm conversion)
- Option C: Add explicit scaling factor

### Step 3: Add QR Code
1. Add QR code generation function (using QRCode library or similar)
2. Add QR code field to CONFIG in both generators:
```javascript
qrCode: { x: 80, y: 80, width: 15, height: 15 } // Example position
```
3. Add `_drawQRCode()` helper function
4. Call QR code drawing in render functions when data is present
5. Update download/preview functions to accept QR data in student/marksheet objects
